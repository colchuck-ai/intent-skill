# Workflow: Product root

Procedure for creating or revising **`docs/product/README.md`**. Paths: [`../assets/CONVENTIONS.md`](../assets/CONVENTIONS.md). Pick **one** product root template: [`../assets/product-root-simple.md`](../assets/product-root-simple.md) (default) or [`../assets/product-root-jtbd.md`](../assets/product-root-jtbd.md) (job-centered outline—omit unused sections per template). See [`../SKILL.md`](../SKILL.md) defaults. **Do not mix** product-root styles in one coherent product tree.

## Draft session

1. **Read [`context-loading.md`](context-loading.md) first** — principles; **B0–B5** definitions.
2. **Load:** **B0** — intent skill slice for this workflow (roles, emit rules, steps)—[`../SKILL.md`](../SKILL.md) as needed; **B1** — product root template ([`product-root-simple.md`](../assets/product-root-simple.md) or [`product-root-jtbd.md`](../assets/product-root-jtbd.md)); **JTBD:** also [`jtbd-primer.md`](jtbd-primer.md); **B3** — parent charter or repo context; **B5** — CR/PDR templates + product-root emit scope when you will emit records ([`emit-records.md`](emit-records.md)).
3. **Draft** per template: **simple**—who, value, boundaries, outcomes; **JTBD**—executor / core job (optional step groups + outcome rows), optional related & emotional jobs, optional lifecycle & buyer, boundaries, outcomes index (one-liners)—**omit** sections that add no value.
4. **Emit** per [`emit-records.md`](emit-records.md):
   - **CR:** for **meaningful edits** to **`docs/product/README.md`**—product-root scope (e.g. `docs/product/crs/` with **`PROD-CR`** naming per [`../assets/CONVENTIONS.md`](../assets/CONVENTIONS.md)).
   - **PDR:** under `docs/product/pdrs/` **only** when this session introduces a **new** product/strategic **decision**—not for routine narrative, outcomes-list edits, or wording-only passes.

## AI review session

1. **Read [`context-loading.md`](context-loading.md) first** — principles; **B0–B5** definitions.
2. **Load:** **B0** — intent skill slice—[`../SKILL.md`](../SKILL.md) as needed; **B2** — review criteria ([`review-product.md`](review-product.md), **Product root README**); **B1** — product root template + draft README; **JTBD:** also [`jtbd-primer.md`](jtbd-primer.md); **B5** — CR/PDR templates + product-root scope if emitting this session.
3. **Review** against criteria; incorporate feedback.
4. **Emit** per [`emit-records.md`](emit-records.md):
   - **CR:** if edits from this session (or gaps from draft) are **meaningful** and not yet covered by a CR—product-root scope.
   - **PDR:** if a **new** strategic decision appears while addressing review feedback (or was missed in draft)—emit under `docs/product/pdrs/`; **do not** create a PDR for every review pass.

## Independent review

1. **Read [`context-loading.md`](context-loading.md) first** — principles; **B0–B5** definitions.
2. **Load:** **B0** — intent skill slice—[`../SKILL.md`](../SKILL.md) as needed; **B2** — review criteria ([`review-product.md`](review-product.md), **Product root README**); **B3** — one-screen parent / charter summary; **B5** — CR/PDR templates + scope if emitting this session.
3. **Emit** per [`emit-records.md`](emit-records.md):
   - **CR:** only if the reviewer still makes **meaningful** edits to the README—or if a **CR** was **missing** for earlier work.
   - **PDR:** only if a **new** strategic decision is recognized at this gate—or if a **PDR** was **missing** for an earlier decision.
